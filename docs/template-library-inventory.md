# Template Library Inventory

This document lists the template-system files that are currently in place and
the templates that are candidates for later expansion.

Path style: each path is the full repository-root path. In a local checkout,
prefix it with the checkout root, for example
`C:\GitHub\repo-template\templates\agent\agent.final-response.standard.md`.

## Current System Files

| Path                    | Description                      | Current use / order                                                                                                |
| ----------------------- | -------------------------------- | ------------------------------------------------------------------------------------------------------------------ |
| `templates/README.md`   | Template-system operating guide. | Read first when editing or extending the library. It defines anatomy, category rules, naming, and extension rules. |
| `templates/MANIFEST.md` | Active template manifest.        | Source of truth for active template IDs and categories. Update whenever templates are added, renamed, or retired.  |

## Expansion Rule

Keep the system small and strict. Prefer the current 17 templates, 9 partials,
3 styles, 2 schemas, and 4 examples until repeated use proves that one of the
candidate templates needs to become active.
