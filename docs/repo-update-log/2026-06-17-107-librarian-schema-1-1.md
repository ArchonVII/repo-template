## 2026-06-17 - Librarian schema 1.1 — version, type, source (OKF integration)

- **Issue/PR:** #107 / (pending)
- **Branch:** agent/claude/107-okf-schema-evolution
- **Changed paths:** scripts/wiki/lib.mjs, scripts/wiki/doctor.mjs, scripts/wiki/lint.mjs, scripts/wiki/lib.test.mjs, docs/LIBRARIAN.md
- **What changed:** Backward-compatible schema bump (1.0 → 1.1) folding in three ideas from Google's Open Knowledge Format (OKF): (1) a declared `SCHEMA_VERSION` in `lib.mjs`, surfaced by `wiki:doctor` as an informational drift signal so a repo inheriting a lagging snapshot can detect how far behind it is (modeled on OKF `okf_version`); (2) an optional page `type` frontmatter key — a routing/filtering axis with a RECOMMENDED-not-closed vocabulary (`TYPE_VALUES`); `wiki:lint` warns, never errors, on an out-of-set value; (3) an optional `source` provenance pointer that pairs with `confidence`, which `wiki:lint` resolves and warns (never errors) on a broken local target (modeled on OKF `resource`). Also documents the `<major>.<minor>` versioning scheme + a changelog, and a "reads open / authoring fails closed" robustness note, in `docs/LIBRARIAN.md`. Every new key is optional — no inherited repo's lint starts failing.
- **Verification:** `npm run wiki:doctor` (schema v1.1, 23 checks pass), `npm run wiki:lint` (4 pages, no errors), `node --test scripts/wiki/lib.test.mjs` (6/6 pass), plus an integration check of `source` resolution (broken target → warn, valid `[[LIBRARIAN]]` → silent). The 2 repo-wide test failures (`checkHookSyntax`) are pre-existing on clean `main` and unrelated to this lane (logged to anomalies).
- **Propagation:** repo-template `main` only. Distribution to consumers rides the gated archon-setup pin bump (archon-setup#257); the additive + version-gated design adds zero new rollout risk. Companion PR (`wiki:graph` visualization) is also under #107.
