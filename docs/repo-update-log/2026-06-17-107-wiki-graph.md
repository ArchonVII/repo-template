- **What changed:** Adds a deterministic `wiki:graph` operation — a *consumer* of the pages tier (no LLM) — that renders the wiki as an interactive HTML graph written to the gitignored `.html-artifacts/wiki-graph.html`. Nodes are colored by `status`; edges are colored by the Librarian's TYPED relations (`depends-on`, `supersedes` with `superseded-by` folded into a single reversed edge, `relates`, and `contradicts` flagged in red) — body links appear as faint untyped edges, suppressed when a typed edge already joins the pair. This is the payoff of typed edges that OKF's untyped graph cannot draw. The renderer is Cytoscape.js loaded from a CDN; the graph DATA is embedded, so only the renderer needs network. `buildModel` is a pure, unit-tested function. Wired into `WIKI_COMMANDS`, `package.json`, the `AGENTS.md` ops list, and the `LIBRARIAN.md` operations table (now five deterministic ops).
- **Verification:** `npm run wiki:graph` (4 pages, 12 edges; artifact written), `npm run wiki:doctor` (25 checks pass — +2 for the new command's script+file checks), `npm run wiki:lint` (4 pages, no errors), `node --test scripts/wiki/lib.test.mjs scripts/wiki/graph.test.mjs` (9/9 pass).
